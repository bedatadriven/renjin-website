Requirements

Renjin will run on any platform with a JVM installed. Therefore, if you want to run Renjin, you need to have installed at least version 7 (or greater) of the Java SE Runtime Environment (JRE). If you haven't, then download and install it from http://www.oracle.com/technetwork/java/javase/downloads/index.html

Running the GUI

The Swing-based Studio GUI is a very simple demo for the moment. Download the JAR file and double-click to execute, or start with the following command:

java -jar renjin-studio-{{ site.renjin-current }}-jar-with-dependencies.jar

where {{ site.renjin-current }} should be replaced with the version number.

Installing the .deb Package

The Debian package can be installed on Ubuntu Linux using the following command:

sudo dpkg -i renjin-debian-package-{{ site.renjin-current }}.deb

After this, you can start Renjin by simply typing renjin on your command line.

Installing via apt

You can also add the Renjin repository to your system to get automatic updates of the Renjin command line tools on Debian and Ubuntu systems.

Create a file called /etc/apt/sources.list.d/renjin.list with the content:

deb https://nexus.bedatadriven.com/content/repositories/renjin-release/ ./

Then run:

sudo apt-get update
sudo apt-get install renjin

Adding to a Maven Project

Using a a modern build tool such a Maven is definitely recommended, particularly if you're planning on including R packages which often have several levels of transitive dependencies.

You can add Renjin to your Maven project by adding the following to your pom.xml file:

<dependencies>
  <dependency>
    <groupId>org.renjin</groupId>
    <artifactId>renjin-script-engine</artifactId>
    <version>RELEASE</version>
  </dependency>
</dependencies>
<repositories>
  <repository>
    <id>bedatadriven</id>
    <name>bedatadriven public repo</name>
    <url>https://nexus.bedatadriven.com/content/groups/public/</url>
  </repository>
</repositories>

Read more about using Renjin in a Java application in the documentation.
